# NexusTelecom - Sistema de Gestión de Servicios de Telecomunicaciones

---

## Resumen del Proyecto

**NexusTelecom** es una plataforma web integral para la gestión operacional de una empresa de telecomunicaciones en Argentina. El sistema permite administrar clientes, equipos, facturación automática, gestión de pagos y reportes financieros en tiempo real.

La aplicación fue desarrollada como monorepo full-stack con el objetivo de centralizar todas las operaciones del negocio en una única solución digital, eliminando procesos manuales y papelasa.

## Arquitectura del Sistema

### Monorepo

```
/
├── apps/
│   ├── backend/          # API REST NestJS
│   └── frontend/        # App Next.js 15
├── package.json        # Workspace raíz
└── turbo.json         # Configuración Turborepo
```

### Stack de Arquitectura

- **API Layer**: NestJS con controladores RESTful
- **Data Layer**: TypeORM con entidades migradas a MySQL
- **Auth Layer**: Firebase Identity + JWT Bearer
- **Presentation Layer**: Next.js 15 App Router (client-side rendering)
- **State Layer**: Zustand (cliente) + React Query (servidor)

### Flujo de Autenticación

1. Usuario se autentica con Firebase (Google/Email)
2. Frontend recibe JWT de Firebase
3. JWT se envía al backend en header `Authorization: Bearer <token>`
4. FirebaseStrategy valida token con Firebase Admin SDK
5. UsersService sincroniza usuario a la base de datos
6. Decorador `@CurrentUser()` inyecta datos del usuario en controllers

## Base de Datos

### Entidades Principales (TypeORM)

- `User` - Usuarios del sistema
- `Cliente` - Clientes del negocio
- `Equipo` - Equipos disponibles
- `DetalleEquipo` - Especificaciones
- `Factura` - Facturas emitidas
- `Pago` - Pagos registrados
- `ClientWallet` - Deuda por cliente
- `WalletTransaction` - Historial transactal
- `Provincia` / `Ciudad` - Geografía argentina

### Convenciones

- Schema: `utf8mb4`
- Timezone: `America/Argentina/Buenos_Aires`
- Columnas: `snake_case` (id_cliente, created_at)
- Propiedades: `camelCase` (idCliente, createdAt)

## Despliegue

### Desarrollo Local

```
docker-compose.yml
├── MySQL (puerto 3306)
├── Backend NestJS (puerto 4000)
└── Frontend Next.js (puerto 3000)
```

### Producción

```
docker-compose.prod.yml
├── MySQL (interno)
├── Backend (interno:4000)
├── Frontend (interno:3000)
└── Nginx (puertos 80/443)
    ├── Sirve frontend estático
    └── Proxea /api/* a backend
```

### CI/CD

- GitHub Actions buildea imágenes
- Imágenespublicadas en GitHub Container Registry
- Despliegue automático al empujar a main
- Health checks post-deploy
- Backups automáticos nocturnos

## Características Distintivas

1. **Monorepo con Turborepo**: Un único repositorio para frontend y backend con builds optimizados
2. **Facturación automática**: Cron job que genera facturas el día 1 de cada mes
3. **Timezone argentina**: Sistema configurado exclusivamente para Argentina
4. **PDFs cliente**: Generación de comprobantes en el navegador con jsPDF
5. **Autenticación robusta**: Capa dual Firebase + JWT Bearer
6. **Estado híbrido**: Zustand + React Query para gestión completa de estado
7. **Docker multi-stage**: Imágenes optimizadas y securizadas
8. **Swagger документация**: API documentada automáticamente
